# Agent Onboarding — www.sqlink.com snapshot

This is a Next.js 16 React app generated by [html-to-react](https://github.com/ariell/html-to-react).
It is a pixel-perfect snapshot of **https://www.sqlink.com** with full SEO + LLM-ready metadata.

## Architecture

```
app/
  [[...slug]]/page.tsx   ← Server Component: exports metadata, renders SnapshotClient
lib/
  SnapshotClient.tsx     ← Client Component: loads CSS+JS in order, injects fragment HTML
  snapshotRegistry.ts    ← Typed registry: maps slug+locale → fragment + css[] + js[] + head
public/
  fragments/             ← Body HTML fragments (one per page × locale)
  _cas/                  ← Content-addressed assets (SHA-256 filenames, immutable)
  llms.txt               ← LLM site index (llmstxt.org spec)
  llms-full.txt          ← Full plain-text content of all pages
  robots.txt             ← AI-crawler-aware robots.txt
  sitemap.json           ← Structured JSON sitemap
  *.md                   ← Markdown mirror of each page (/<slug>.md)
```

## How to add a page

1. Add an entry to `lib/snapshotRegistry.json`:
   ```json
   {
     "slug": "new-page",
     "locale": "en",
     "dir": "ltr",
     "fragment": "fragments/new-page-en-body.html",
     "css": [...],
     "js": [...],
     "loadOrderSha": "...",
     "gsapHook": false,
     "loadedDelayMs": 800,
     "head": { "title": "...", "metaTags": [...], "linkTags": [], "jsonLd": [] }
   }
   ```
2. Add the fragment HTML to `public/fragments/new-page-en-body.html`
3. Run `npm run build` to verify

## How to modify content

- Edit the fragment at `public/fragments/<slug>-<locale>-body.html`
- The fragment is raw HTML body — edit text, swap images, change links directly
- After editing, run `npm run verify:parity` (if configured) to confirm visual parity

## Critical rules (do not violate)

1. **Never change CSS load order** in `snapshotRegistry.ts` — cascade is load-order dependent
2. **Never add `async` or `defer` to scripts in deps** — timing is preserved intentionally
3. **Never remove `data-snapshot-client` attribute** — used by parity verifier
4. **Never inline CAS assets** — they are content-addressed and cache-immutable
5. **Never modify `loadOrderSha`** — it is a verification checksum
